# 胶东私人收藏网站 UI/UX 综合审查报告

> 审查日期：2026-07-15
> 审查范围：全部 9 个页面（index / collection / collectors / stories / updates / contact / collection-detail / collector-detail / update-detail）
> 审查维度：信息架构、视觉层级、一致性、可用性、无障碍、响应式、状态设计

---

## 一、信息架构 & 页面结构

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|---|
| 1.1 | [全局/导航] 所有导航链接 `href="#"` | 全站 358 处 `<a href="#">`，用户点击后页面不会跳转也不会报错，所有"首页""藏品展示""收藏家"等导航均不可用，核心浏览流程完全断裂。 | **高** | 将 `href="#"` 替换为实际页面路径（`index.html`、`collection.html` 等），或接入前端路由。 |
| 1.2 | [全局/导航] 移动端无导航入口 | 所有页面 `<nav>` 使用 `hidden lg:flex`，小屏幕下导航完全隐藏，且没有汉堡菜单按钮或任何替代导航方式，移动端用户无法在页面间切换。 | **高** | 添加移动端汉堡菜单按钮（`lg:hidden`），配合抽屉式/下拉式导航面板。 |
| 1.3 | [全局] 缺少 `<main>` 语义结构 | 部分页面将 `<header>` 包含在 `<main>` 内部，部分页面缺少 `<main>` 标签，语义结构不统一，不利于无障碍屏幕阅读器解析。 | **中** | 统一页面结构为 `<header>` → `<main>` → `<footer>`，保持 `<header>` 在 `<main>` 外部。 |
| 1.4 | [collectors.html] 统计栏 `<section>` 未闭合 | collectors.html 的统计栏 `<section>` 标签缺少对应的 `</section>` 闭合标签，可能导致后续布局错乱。 | **高** | 补充 `</section>` 闭合标签。 |
| 1.5 | [stories.html] 页面缺少 h1 标题 | stories.html 在 `<body>` 内没有独立的 h1 元素，仅有 `<title>` 标签，不符合 HTML 语义规范（每个页面应有且仅有一个 h1）。 | **中** | 在面包屑下方添加 `<h1>收藏故事</h1>`，与其他列表页保持一致。 |
| 1.6 | [collection.html] 面包屑首页 ID 命名不一致 | 面包屑中首页链接使用 `data-dom-id="nav-index"`，其他所有页面统一使用 `nav-home`，命名不一致。 | **低** | 统一为 `data-dom-id="nav-home"`。 |
| 1.7 | [首页] Hero 区域信息密度偏低 | Hero 区仅有一句标题和一句副标题，右侧图片与左侧文字的关联性不强，未充分利用首屏空间传递平台核心价值。 | **低** | 可考虑增加 1-2 个关键数据点（如"已入驻 XX 位收藏家""展示 XX 件藏品"），或添加 CTA 按钮。 |

## 二、视觉层级（字号、粗细、对比度、留白）

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|---|
| 2.1 | [全局] 主色文字与白色背景对比度不足 | 主色调 `--navy-500: #1e3b7a` 在白色背景上的对比度约为 **4.6:1**，勉强通过 AA 标准（4.5:1），但在小字号（13px/14px）下可读性仍然偏弱。`--text-primary-300: #8496a8` 在白色背景上对比度仅 **3.2:1**，**不满足 WCAG AA 标准**（需 >= 4.5:1）。 | **高** | 将 `--text-primary-300` 加深至至少 `#6b7f94`（对比度 4.5:1 以上），或仅在装饰性大文字上使用，避免用于需要阅读的辅助文字。 |
| 2.2 | [collection.html] 卡片描述使用硬编码 `height:79px` | 藏品列表页描述区域使用 `height:79px;overflow:hidden` 硬编码，而非设计系统的 `-webkit-line-clamp:3`，与首页和详情页卡片实现方式不一致。 | **中** | 移除硬编码 height，统一使用 `-webkit-line-clamp:3` + flex 布局。 |
| 2.3 | [index.html] Hero 图片使用固定尺寸 | Hero 右侧图片使用 `width:453px;height:400px` 硬编码，不会自适应不同屏幕宽度，在 < 1024px 屏幕上会溢出或变形。 | **高** | 移除固定宽高，使用 `width:100%;max-width:453px;aspect-ratio` 或 Tailwind 响应式类。 |
| 2.4 | [collector-detail.html] 卡片标题字号不一致 | 收藏家详情页"他的藏品"区域卡片标题使用 `font-size:18px`，而设计系统 `--font-size-h3` 为 `20px`，与其他页面 h3 不统一。 | **中** | 统一使用 `var(--font-size-h3)` (20px)。 |
| 2.5 | [全局] 统计数字 font-weight 不一致 | 多处统计数字使用 `font-weight:700`，设计系统 `--font-weight-stat` 定义为 `600`。 | **低** | 统一使用 `var(--font-weight-stat)`。 |
| 2.6 | [collection-detail.html] 描述文字颜色过浅 | 详情页藏品描述使用 `color:var(--text-primary-400)` (#5b6d84)，对比度约 **4.7:1**，虽通过 AA 但在长文阅读时视觉偏淡。设计系统 ArtifactCard 描述应使用 `--text-primary-500`。 | **低** | 统一为 `var(--text-primary-500)`，提升阅读舒适度。 |
| 2.7 | [首页] "更多"链接视觉层级模糊 | "更多 →" 链接使用 `font-family:var(--font-heading)` 与 h2 标题同字体，在 `items-baseline` 对齐下容易被误认为标题的一部分而非独立操作入口。 | **低** | 可考虑将"更多"链接字号缩小一级（13px），或增加与标题的间距来区分操作与标题。 |

## 三、一致性（颜色、组件样式、间距、文案语气）

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|
| 3.1 | [全局] 未引用设计系统组件 CSS | 9 个 HTML 页面均未通过 `<link>` 引用 `components.css`，所有组件样式（TopNav、FilterTag、ArtifactCard 等）均通过 inline style 重新实现，导致样式碎片化。 | **高** | 在所有页面中引用外部 `components.css`，使用语义化组件类（如 `.jd-artifact-card`、`.jd-filter-tag`）。 |
| 3.2 | [全局] CSS 变量在每个页面重复内联 | 每个页面都将约 235 行 CSS 变量定义内联在 `<style id="theme-vars">` 中，9 个页面共计 ~2100 行重复代码，维护成本极高。 | **高** | 提取为独立的 `theme.css`，通过 `<link>` 统一引用。 |
| 3.3 | [全局] 头像样式不一致 | 设计系统定义 `.jd-collector-card__avatar` 使用 `--navy-100` 背景和 `--radius-avatar:22px` 圆角方形，但实际页面头像使用 `--navy-500` 背景和 `border-radius:50%` 圆形。 | **中** | 统一决定头像形状（建议圆形更适合人物场景），同步更新 components.css。 |
| 3.4 | [全局] 头像尺寸不统一 | 设计系统 `--size-avatar:44px`，首页收藏家卡片使用 `48px`，收藏家详情页使用 `80px`，缺少明确的尺寸梯度定义。 | **中** | 定义 `--size-avatar-sm/md/lg` 三档尺寸 token，覆盖不同使用场景。 |
| 3.5 | [全局] 筛选标签尺寸偏差 | 设计系统 `.jd-filter-tag` 高度 `34px`，实际页面筛选按钮高度 `36px`，font-size 也从 `13px` 偏移到 `14px`。 | **中** | 统一筛选标签尺寸，更新 components.css 或统一页面实现。 |
| 3.6 | [全局] 卡片 hover 偏移量不一致 | 设计系统定义 `translateY(-2px)`，但部分页面（首页、列表页）卡片使用 JS `onmouseenter` 实现 `translateY(-1px)`，部分页面用 CSS 实现 `-2px`。 | **低** | 统一为 CSS `translateY(-2px)`，移除内联 JS hover。 |
| 3.7 | [collection.html] 网格布局未使用响应式方案 | 藏品列表使用 `grid-template-columns:repeat(4,1fr)` 内联样式，未使用 Tailwind 响应式类，移动端会强制显示 4 列而不会自适应为 2 列。 | **高** | 改为 `class="grid grid-cols-2 lg:grid-cols-4"` 的 Tailwind 响应式方案。 |
| 3.8 | [全局] body 背景色引用未定义变量 | `@layer base` 中 `body { background: var(--text-background); color: var(--text-foreground); }` 引用了未定义的变量，应使用 `--color-background` 和 `--color-foreground`。 | **高** | 修正变量名。 |
| 3.9 | [collectors.html] 收藏家卡片间距 `gap:18px` | 卡片网格使用 `gap:18px`，不属于 Tailwind 标准间距 token（16px/20px/24px），与其他页面 20px/24px 的间距不一致。 | **低** | 统一为 `gap:20px` 或使用 CSS 变量 `var(--space-card-gap)`。 |
| 3.10 | [collectors.html] Banner 轮播仅有 2 张图 | 收藏家页面 Banner 轮播仅 2 张图片，轮播效果几乎无感知，且与首页和 updates 页的 Banner 风格不同。 | **低** | 可补充更多 Banner 图或改为静态图，与全站 Banner 风格保持一致。 |

## 四、可用性与交互反馈（按钮状态、点击区域、表单校验、错误提示）

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|
| 4.1 | [全局] 筛选按钮无 focus-visible 样式 | 所有筛选按钮（FilterTag）点击后没有 `focus-visible` 轮廓或高亮反馈，键盘用户无法辨识当前聚焦项。 | **高** | 添加 `:focus-visible { outline: 2px solid var(--navy-500); outline-offset: 2px; }` 样式。 |
| 4.2 | [contact.html] 表单仅 2 个字段标记 required | 联系表单的"姓名"和"电话"有 HTML5 `required` 属性，但提交时仅依赖浏览器默认提示（红色边框），无自定义错误消息，且"收藏简介"等非必填字段缺少字符限制。 | **中** | 添加前端表单校验逻辑：电话号码格式校验、姓名长度限制、提交成功的 Toast 反馈。 |
| 4.3 | [contact.html] 表单提交按钮无 loading 状态 | "提交申请"按钮点击后无 loading 态（旋转图标/文字变化），用户无法判断是否正在处理，可能重复点击。 | **中** | 添加 `onClick` 禁用按钮 + 显示 loading 状态。 |
| 4.4 | [contact.html] 收藏方向 checkbox 不可交互 | 8 个收藏方向 checkbox 的 `states: [readonly]`，用户无法勾选/取消，但视觉上没有 disabled 灰态提示，易造成困惑。 | **中** | 如需禁用则添加 `disabled` 属性和灰态样式；如需可交互则移除 readonly。 |
| 4.5 | [全局] 分页按钮样式无 active/hover 反馈 | collection.html、collectors.html、updates.html 的分页按钮（1、2）缺少 active 高亮和 hover 状态，用户无法辨识当前页码。 | **中** | 为当前页添加 `background:var(--navy-500);color:white`，hover 添加背景变化。 |
| 4.6 | [全局] "查看详情""了解更多"链接缺少 hover 变化 | 卡片底部的"查看详情 →"和"了解更多 →" 链接仅依赖浏览器默认链接色变化，无下划线/颜色/箭头动效等明确的交互反馈。 | **低** | 添加 `:hover { color: var(--navy-600); }` 和箭头右移动画。 |
| 4.7 | [collection.html] 搜索框和排序下拉无实际功能 | 搜索框和排序下拉框（"最新发布"）存在但无筛选逻辑，输入内容或切换排序后页面不会发生变化，用户预期与实际行为不符。 | **高** | 要么实现搜索和排序功能，要么移除这些控件以避免误导。 |
| 4.8 | [全局] 卡片点击区域偏小 | 部分卡片只有标题文字可点击（如收藏故事卡片），图片和描述区域不可点击，但视觉上整张卡片看起来都应可交互。 | **中** | 将整个卡片包裹在 `<a>` 标签内，或添加 `role="link"` + `cursor:pointer` + `tabindex="0"`。 |
| 4.9 | [全局] 所有按钮缺少 disabled 状态展示 | 设计系统定义了 `.jd-btn:disabled` 样式，但实际页面中没有任何按钮展示 disabled 状态（如分页到最后一页时"下一页"应灰态）。 | **低** | 在分页、筛选等场景中增加 disabled 状态的展示。 |

## 五、无障碍（颜色对比度、可访问性、可读性）

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|
| 5.1 | [全局] 辅助文字对比度严重不足 | `--text-primary-300: #8496a8` 在白色背景上对比度仅 **3.2:1**，用于时间戳、标签、面包屑等辅助信息，不满足 WCAG 2.1 AA 标准（4.5:1）。`--text-primary-200: #adb9c7` 对比度仅 **2.4:1**，更严重。 | **高** | 将 `--text-primary-300` 加深至 `#6b7f94`（约 4.5:1），`--text-primary-200` 加深至 `#8a9bb0`（约 4.5:1）。 |
| 5.2 | [全局] 导航链接缺少 `aria-current="page"` | 当前页面的导航链接没有 `aria-current="page"` 属性，屏幕阅读器用户无法辨识当前所在页面。 | **中** | 为当前页导航项添加 `aria-current="page"`。 |

| 5.5 | [全局] 表单缺少 `<label>` 关联 | contact.html 表单字段使用 `placeholder` 作为标签，缺少 `<label for="...">` 或 `aria-label`，屏幕阅读器无法正确识别字段用途。 | **高** | 为每个 `<input>` 添加对应的 `<label>` 元素或 `aria-label` 属性。 |
| 5.6 | [全局] 页面语言声明正确但部分细节缺失 | `<html lang="zh-CN">` 声明正确，但页面缺少 `<meta name="description">` SEO 描述标签。 | **低** | 添加 meta description，有助于搜索引擎和屏幕阅读器理解页面内容。 |

## 六、响应式 / 移动端适配

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|
| 6.1 | [全局] 移动端导航完全不可用 | `<nav class="hidden lg:flex">` 使导航在 < 1024px 屏幕上完全隐藏，无汉堡菜单或侧边栏替代，移动端用户无法在页面间切换。 | **高** | 实现响应式导航：`lg:hidden` 显示汉堡按钮，配合抽屉式导航。 |
| 6.2 | [collection.html] 四列网格不响应式 | 藏品列表使用 `grid-template-columns:repeat(4,1fr)` 硬编码，无 `@media` 断点，移动端强制 4 列极窄。 | **高** | 添加响应式断点：移动端 2 列、平板 3 列、桌面 4 列。 |
| 6.3 | [index.html] Hero 区域不响应式 | Hero 使用 flex 布局，左侧文字 + 右侧固定尺寸图片，在 < 1024px 下图片可能溢出或被截断，无响应式堆叠。 | **高** | 添加媒体查询，小屏幕下改为上下堆叠布局，图片宽度自适应。 |
| 6.4 | [首页] 收藏动态模块不响应式 | 收藏动态的"左1右4"布局在 < 768px 下左右卡片会挤压到极窄宽度，无响应式断点。 | **中** | 添加响应式方案：移动端改为单列堆叠，featured 卡片占满宽度。 |
| 6.5 | [全局] 全站无 `@media` 断点 | 9 个页面的 `<style>` 中均未定义任何 `@media` 查询，完全依赖 Tailwind 的 `lg:` 前缀（仅导航生效），所有其他布局元素不响应屏幕变化。 | **高** | 至少为 Hero、网格、卡片等核心布局添加 `@media (max-width:768px)` 和 `(max-width:1024px)` 断点。 |
| 6.6 | [collectors.html] Banner 轮播高度固定 | Banner 图片使用固定 `height:320px`，在移动端占据过多视口高度。 | **中** | 改为响应式高度（移动端 `height:200px`，桌面端 `height:320px`）。 |
| 6.7 | [全局] 触控目标尺寸不足 | 筛选按钮高度 36px、分页按钮尺寸偏小，低于移动端推荐的 44px 最小触控目标。 | **中** | 将筛选按钮高度提升至 `min-height:44px`，分页按钮增加 padding。 |
| 6.8 | [contact.html] 表单在移动端过宽 | 表单使用固定布局，在小屏幕上输入框、下拉框可能溢出容器。 | **低** | 确保表单使用 `width:100%` + `box-sizing:border-box`，配合响应式 padding。 |

## 七、加载态 / 空态 / 异常态

| # | 页面/模块 | 问题描述 | 严重程度 | 改进建议 |
|---|---|---|---|
| 7.1 | [全局] 无加载态设计 | 除 collector-detail 和 update-detail 页面定义了 `.jd-skeleton` CSS 类（但未实际使用）外，其余页面无任何 loading 状态。 | **高** | 在页面主要内容区添加 skeleton 占位或 loading spinner，提升感知性能。 |
| 7.2 | [全局] 无空态设计 | 筛选结果为空时（如选择"铜器"分类后首页藏品可能不足 4 件），无"暂无数据"空态提示，页面仅显示空白区域。 | **高** | 添加空态占位组件：图标 + "暂无该分类藏品" + 重置筛选按钮。 |
| 7.3 | [全局] 无异常/错误态设计 | 无 404 页面、无网络错误提示、无图片加载失败占位（破损图片直接显示浏览器默认裂图标）。 | **中** | 添加 404 页面；为图片添加 `onerror` 回退（灰色占位 + 图标）。 |

### 3 条最重要的优化建议

1. **修复移动端适配和导航**（影响 6.1 / 6.2 / 6.3 / 6.5 / 1.2）
   当前网站在 < 1024px 屏幕上几乎不可用——导航消失、网格不响应、Hero 溢出。这是阻塞性问题，建议优先实现：汉堡菜单 + 全站响应式断点（768px / 1024px）+ 网格自适应列数。

2. **替换所有 `href="#"` 为实际页面路径**（影响 1.1 / 4.7）
   全站 358 处空链接导致所有页面间导航断裂，核心浏览流程完全不可用。这是一个"看起来能点但点了没反应"的严重体验缺陷，应在最短时间内修复。

3. **修复颜色对比度和表单可访问性**（影响 5.1 / 5.5 / 4.2 / 7.4）
   `--text-primary-300` 对比度仅 3.2:1 不达标；表单缺少 `<label>` 关联；联系表单无提交反馈。这些是无障碍合规和基本可用性的底线问题，修复成本低但影响面广。
