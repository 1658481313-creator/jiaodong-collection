## 一、样式系统总则

### 1.1 设计哲学

**信息平台而非艺术画廊**。深海军蓝 + 无衬线中文 + 信息密集型布局 = 理性、克制、条理清晰的信息展示风格。

| 原则 | 含义 | 工程体现 |
|------|------|----------|
| 克制 | 不做装饰性视觉表达 | 默认态无阴影、无渐变、无纹理 |
| 一致 | 同类元素视觉表现完全统一 | 所有 Token 值通过 CSS Variables 统一引用 |
| 信息优先 | 每个模块提供可阅读的信息量 | 不做纯视觉装饰区，每个区块有标题+描述+数据 |
| 可迭代 | 样式变更不影响业务逻辑 | Token 层与组件层分离，改 Token 不改组件代码 |

### 1.2 三条硬规则

1. **色调不变** — `#1e3b7a` 为主色锚点，不做暖色系扩展
2. **纯中文** — 不使用英文标题、双语排版
3. **信息优先** — 不做纯视觉装饰区

### 1.3 样式系统分层架构

```
┌─────────────────────────────────────────────────┐
│  Layer 5: 页面样式 (views/*.vue <style scoped>) │  ← 页面级组装
├─────────────────────────────────────────────────┤
│  Layer 4: 区段样式 (sections/*.vue <style>)     │  ← 区段级组合
├─────────────────────────────────────────────────┤
│  Layer 3: 组件样式 (components/*.vue <style>)   │  ← 组件级样式
├─────────────────────────────────────────────────┤
│  Layer 2: SCSS Mixin (_mixins.scss)             │  ← 样式模板
├─────────────────────────────────────────────────┤
│  Layer 1: Design Token (variables.css :root)    │  ← 设计令牌
├─────────────────────────────────────────────────┤
│  Layer 0: CSS Reset (_reset.scss)               │  ← 基础重置
└─────────────────────────────────────────────────┘
```

**依赖规则**：上层引用下层，下层不感知上层。  
- Layer 0 → 独立，无依赖  
- Layer 1 → 依赖 Layer 0（Reset 后定义变量）  
- Layer 2 → 依赖 Layer 1（Mixin 内引用 CSS Variables）  
- Layer 3-5 → 依赖 Layer 1 + Layer 2（组件样式引用 Variables + Mixin）  

### 1.4 样式文件组织

```
shared/tokens/
├── variables.css          # :root CSS Variables（运行时可变，主题切换基础）
├── variables.scss         # SCSS 变量映射（编译时常量）
├── _tokens.scss           # SCSS @import 入口（汇总 _mixins + _reset）

└── _reset.scss            # CSS 基础重置

src/assets/styles/ (PC 端)
├── variables.css          # 从 shared/tokens 同步
├── _tokens.scss           # 从 shared/tokens 同步
├── _mixins.scss           # 从 shared/tokens 同步
├── _reset.scss            # 从 shared/tokens 同步
├── _typography.scss       # 字体样式类（.text-h1, .text-body 等）
├── _layout.scss           # 布局类（.container, .card-grid 等）

```

## 二、Token 架构

### 2.1 Token 命名规范

所有 Token 遵循 `--{category}-{name}` 双横线命名法：

| 前缀 | 类别 | 示例 |
|------|------|------|
| `--c-` | 色彩 (Color) | `--c-primary`, `--c-text` |
| `--bg-` | 背景色 (Background) | `--bg-band`, `--bg-hero-note` |
| `--art-` | 藏品分类色 (Art) | `--art-blue`, `--art-jade` |
| `--border-` | 边框色 (Border) | `--border-warm`, `--border-blue` |
| `--fs-` | 字号 (Font Size) | `--fs-body`, `--fs-button` |
| `--fw-` | 字重 (Font Weight) | `--fw-bold`, `--fw-regular` |
| `--lh-` | 行高 (Line Height) | `--lh-title`, `--lh-body` |
| `--gap-` | 间距 (Gap) | `--gap-grid`, `--gap-pills` |
| `--pad-` | 内边距 (Padding) | `--pad-card`, `--pad-section` |
| `--radius-` | 圆角 (Radius) | `--radius-card`, `--radius-button` |
| `--transition-` | 过渡 (Transition) | `--transition-fast`, `--transition-normal` |
| `--z-` | 层级 (Z-index) | `--z-header`, `--z-modal` |
| `--page-` | 页面参数 (Page) | `--page-width`, `--page-content-width` |
| `--nav-` | 导航参数 (Nav) | `--nav-height`, `--nav-gap` |

### 2.3 Token 使用优先级

```
开发者编写样式时的决策树：

1. 是否有对应 Token？
   ├── 有 → 直接使用 var(--xxx)
   └── 无 → 进入第 2 步

2. 是否可以复用已有 Token 的组合？
   ├── 可以 → 组合使用（如 padding: var(--pad-section-y) var(--page-side-padding)）
   └── 不可以 → 进入第 3 步

3. 是否需要新增 Token？
   ├── 该值会被 3+ 处使用 → 新增 Token（需走 Token 变更流程）
   └── 仅 1-2 处使用 → 硬编码值，加注释说明
```

## 四、色彩系统

### 4.1 色彩语义映射

每个色彩 Token 有明确的语义角色，不可混用：

| Token | 语义角色 | 可用于 | 不可用于 |
|-------|----------|--------|----------|
| `--c-primary` | 品牌/强调 | 按钮背景、链接文字、标签文字、头像背景、竖条、激活态指示器 | 正文文字、卡片背景、边框 |
| `--c-text` | 主标题文字 | H1/H2/H3、卡片标题、品牌名、Note 标题 | 描述文字、按钮文字、背景色 |
| `--c-text-secondary` | 辅助/描述文字 | 正文描述、副标题、时间戳、导航非激活态、免责声明 | 标题文字、按钮文字 |
| `--c-white` | 背景/反色文字 | 页面背景、卡片背景、按钮文字（在主色底上） | 标题文字、边框色 |
| `--bg-band` | 区段交替底色 | StatsBand、轮播右栏、分类浏览区段底色 | 卡片背景、按钮背景 |
| `--border-warm` | 暖灰边框 | Header 底线、Footer 顶线、藏品卡片边框 | Stat 卡片边框、动态卡片边框 |
| `--border-blue` | 蓝灰边框 | Stat 卡片边框、分类 Pill 边框 | 藏品卡片边框 |
| `--border-gray` | 灰色边框 | 动态卡片边框、轮播分隔线 | 藏品卡片边框 |

### 4.2 色彩使用规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 主色仅用于强调 | `#1e3b7a` 不做大面积背景（除头像/竖条等小面积元素） | ✅ 按钮背景 ✘ 页面背景 |
| 文字色二选一 | 标题用 `--c-text`，描述用 `--c-text-secondary`，不使用中间值 | ✅ `color: var(--c-text)` ✘ `color: #333` |
| 区段底色交替 | 白底与 `--bg-band` 交替，形成视觉节奏 | 白→蓝→白→蓝→白 |
| 分类色彩编码 | 每个藏品分类使用专属占位色 | 瓷器→`--art-blue`，书画→`--art-ink` |
| 深底浅字例外 | 书画类视觉区用 `--art-ink` 深蓝底时，文字须为白色 | `color: var(--c-white)` on `background: var(--art-ink)` |

### 4.3 可访问性对比度矩阵

| 前景色 | 背景色 | 对比度 | WCAG AA | WCAG AAA | 使用场景 |
|--------|--------|--------|---------|----------|----------|
| `#15233a` | `#ffffff` | 15.3:1 | ✅ | ✅ | 主标题/品牌名 |
| `#5b6d84` | `#ffffff` | 4.6:1 | ✅ | ✘ | 正文描述（4.5:1 门槛） |
| `#1e3b7a` | `#ffffff` | 8.4:1 | ✅ | ✅ | 链接/标签文字 |
| `#ffffff` | `#1e3b7a` | 8.4:1 | ✅ | ✅ | 按钮文字/头像文字 |
| `#ffffff` | `rgba(20,35,58,0.85)` | ≈10:1 | ✅ | ✅ | 轮播遮罩文字 |
| `#1e3b7a` | `#F2F9FC` | 7.8:1 | ✅ | ✅ | 浅蓝底上的主色文字 |
| `#5b6d84` | `#F2F9FC` | 4.3:1 | ✅* | ✘ | 浅蓝底上的辅助文字（大字号 OK） |

> *`#5b6d84` on `#F2F9FC` 对比度 4.3:1，略低于 4.5:1 标准。在浅蓝底区段上，辅助文字应使用 `--c-text`（`#15233a`）替代 `--c-text-secondary` 以确保达标。

## 八、组件样式架构

### 8.1 BEM 命名规范

所有组件 CSS 类名遵循 BEM (Block Element Modifier) 命名法：

```
.block {}                    → 组件根元素
.block__element {}           → 组件内部子元素
.block--modifier {}          → 组件状态/变体
.block__element--modifier {} → 子元素状态/变体
```

**命名规则**：
- Block: 横线连接，语义化（`collection-card`, `primary-btn`, `pill-tag`）
- Element: 双下划线连接（`collection-card__image`, `collection-card__title`）
- Modifier: 双横线连接（`pill-tag--active`, `btn--disabled`）

**示例**：
```scss
// CollectionCard.vue
.collection-card {
  &__image { ... }           // 图片视觉区
  &__body { ... }            // 卡片内容区
  &__title { ... }           // 卡片标题
  &__desc { ... }            // 卡片描述
  &__tag { ... }             // 卡片标签
  &--hover { ... }           // hover 状态
  &--clickable { ... }       // 可点击状态
}
```

### 8.2 Scoped 样式规则

```vue
<!-- ✅ 正确：所有组件使用 scoped -->
<style scoped lang="scss">
.collection-card {
  @include card-base;
  @include card-border(var(--border-warm));
  @include card-hover;

  &__title {
    font-size: var(--fs-card-h3);
    color: var(--c-text);
  }
}
</style>

<!-- ✘ 错误：不使用 scoped，样式污染全局 -->
<style lang="scss">
.collection-card { ... }
</style>
```

**`:deep()` 穿透规则**：仅在需要修改子组件内部样式时使用：
```scss
// 父组件修改子组件样式
.featured-section {
  // 正常情况下通过 Props 传递，不直接修改子组件样式
  // 仅在无法通过 Props 控制时使用 :deep()
  :deep(.collection-card__title) {
    font-size: var(--fs-card-h3);  // 确保子组件标题字号一致
  }
}
```

| 层级 | Token | 元素 | 视觉表现 |
|------|-------|------|----------|
| L0 | — | Section 白底 / `--bg-band` 底 | 纯色底，无阴影 |
| L1 | — | 卡片 | 1px 边框 + 8px 圆角 + 白底 |
| L2 | `--shadow-card-hover` | 卡片 hover | `translateY(-2px)` + shadow |
| L3 | — | Header / Footer | 1px 边框线 + 半透白 bg + backdrop-blur |
| L4 | `--bg-carousel-mask` | 轮播遮罩条 | `rgba(20,35,58,0.85)` 深色条 |
| L5 | — | 头像 | `--c-primary` 实色底 + 22px 圆角 |
| L6 | — | 竖条指示器 | `--c-primary` 3px 宽实色条 |

### 9.2 阴影规则

**默认态无阴影**。仅在 hover 态使用阴影：

```scss
// 卡片 hover 阴影
.card {
  // 默认态：无阴影
  transition: all var(--transition-normal);

  &:hover {
    transform: translateY(-2px);
    box-shadow: var(--shadow-card-hover);  // 0 4px 12px rgba(0,0,0,0.08)
  }
}

// 按钮 hover 阴影
.btn-primary {
  &:hover:not(:disabled) {
    transform: translateY(-1px);
    box-shadow: var(--shadow-button-hover);  // 0 4px 12px rgba(0,0,0,0.15)
  }
}
```

### 9.3 禁止使用的视觉效果

| 禁止项 | 原因 | 例外 |
|--------|------|------|
| 默认态阴影 | 增加视觉噪音，破坏克制风格 | 无 |
| 渐变 | 不符合纯色克制风格 | 仅轮播图片占位区允许 |
| 毛玻璃 | 除 Header/Footer 外不使用 | Header/Footer 的 backdrop-blur |
| 纹理/图案 | 增加视觉噪音 | 无 |
| text-shadow | 干扰文字可读性 | 无 |
| letter-spacing | 破坏中文字体自然间距 | 无 |
| italic | 不符合正式信息平台风格 | 无 |

## 十、动效系统

### 10.1 过渡时长

| Token | 时长 | 用途 |
|-------|------|------|
| `--transition-fast` | 150ms ease | 按钮 hover、Pill 切换、导航高亮、标签切换 |
| `--transition-normal` | 300ms ease | 卡片 hover 上浮、轮播切换 opacity、模态框进出 |
| `--transition-slow` | 500ms ease | 页面区域渐显（仅首屏加载动画） |

### 10.2 交互状态定义

| 元素 | Default | Hover | Active/Focus | Disabled |
|------|---------|-------|-------------|----------|
| PrimaryBtn | bg `--c-primary`, text white | bg `--c-primary-hover`, translateY(-1px), shadow | outline 2px `--c-primary` | opacity 0.6 |
| GhostBtn | bg white, border+text `--c-primary` | bg `--c-primary`, text white | outline 2px `--c-primary` | opacity 0.6 |
| 卡片 | 无阴影 | translateY(-2px), shadow `--shadow-card-hover` | — | — |
| PillTag | bg white, text `--c-primary` | bg `--c-primary`, text white | — | — |
| 导航链接 | `--c-text-secondary` Regular | `--c-primary` | Bold + `--c-primary`（当前页） | — |
| 更多链接 | 无下划线 | 下划线 | — | — |
| 轮播图片 | 显示首帧 | 暂停自动播放 | — | — |

### 10.3 轮播动效

```scss
// 轮播切换过渡
.carousel-slide {
  transition: opacity var(--transition-carousel);
  
  &--active {
    opacity: 1;
  }
  
  &--inactive {
    opacity: 0;
    position: absolute;
    pointer-events: none;
  }
}

// 轮播指示器过渡
.carousel-dot {
  transition: all var(--transition-fast);
  
  &--active {
    width: 8px;
    height: 8px;
    background: var(--c-primary);
  }
  
  &--inactive {
    width: 6px;
    height: 6px;
    background: var(--c-inactive-dot);
  }
}

// 轮播竖条过渡
.carousel-bar {
  transition: height var(--transition-normal), background var(--transition-fast);
  
  &--active {
    height: 44px;
    background: var(--c-primary);
  }
  
  &--inactive {
    height: 32px;
    background: var(--c-inactive-bar);
  }
}
```

### 10.4 动效原则

| 原则 | 说明 |
|------|------|
| 快速反馈 | 按钮和标签的 hover 过渡用 150ms，确保即时反馈 |
| 平滑过渡 | 卡片和轮播的过渡用 300ms，确保视觉流畅 |
| 不滥用动效 | 不做入场动画、不做滚动视差、不做骨架屏抖动 |
| 尊重 prefers-reduced-motion | 用户设置减少动效时，禁用所有过渡 |

```scss
// 尊重用户动效偏好
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 十二、主题系统

### 12.1 当前主题：Light（默认）

项目当前只有 Light 主题，所有 Token 值即为 Light 主题值。

### 12.2 Dark 主题预留架构

未来如需支持 Dark 主题，通过 `[data-theme="dark"]` 属性覆盖 CSS Variables：

```css
/* 主题切换示例（预留，当前不实现） */
[data-theme="dark"] {
  /* 基础色板反转 */
  --c-white: #0f172a;                /* 页面背景 → 深色 */
  --c-text: #f1f5f9;                 /* 主文字 → 浅色 */
  --c-text-secondary: #94a3b8;       /* 辅助文字 → 浅灰 */

  /* 主色微调（暗色背景下主色需提亮） */
  --c-primary: #3b6fd4;              /* 深蓝提亮 */
  --c-primary-hover: #4d80e0;
  --c-primary-press: #2a5cc0;

  /* 区段背景 */
  --bg-band: #1a2740;                /* 浅蓝底 → 深蓝灰底 */
  --bg-hero-note: #1e2d4a;

  /* 边框色 */
  --border-warm: #2a3a55;
  --border-blue: #2d4060;
  --border-gray: #2a3a50;

  /* Header */
  --bg-header: rgba(15, 23, 42, 0.96);
}
```

### 12.3 主题切换原理

```
CSS Variables 是运行时可变的：
1. :root 定义默认值（Light 主题）
2. [data-theme="dark"] 覆盖特定变量
3. 所有组件引用 var(--xxx)，自动跟随主题变化
4. 组件代码零修改，只需切换 <html data-theme="dark">
```

**前提**：所有组件样式必须使用 `var(--xxx)` 引用 Token，不能硬编码色值。这是未来支持主题切换的基础。

## 十五、迭代规范

### 15.1 Token 变更流程

> 权威定义见 18.4 节（含 Design Library 同步步骤）。以下为简述。

1. 提出变更 → Issue 中描述，标注 `design-token-change`
2. 评估影响 → 全局搜索 `var(--xxx)` 确认引用处数量
3. 修改 Token → 同步修改 Design Library (`colors_and_type.css` + `css.json`)、工程层 (`variables.css` + `variables.scss`)、文档
4. 同步两端 → PC 端 `npm update`，UniApp 端手动拷贝
5. 验证 → Design Library 预览走查 + 前端走查 + 可访问性检测
6. 发布 → PR 关联 Issue，Design Library `metadata.json` 版本号 +1

### 15.2 新增 Token 规则

| 规则 | 说明 |
|------|------|
| 命名 | 遵循 `--{category}-{name}` 双横线命名法 |
| 分类 | 归入正确的 Token 分类（色彩/字号/间距/圆角/动效） |
| 文档 | 在《视觉风格文档》和本文档同步添加说明 |
| 值来源 | 必须有画布实测数据或设计稿明确标注 |
| 3 次规则 | 该值将被 3+ 处使用才新增 Token，否则硬编码 |

### 15.3 废弃 Token 规则

```
Token 废弃流程：

1. 标记废弃
   → 在 variables.css 中添加注释：/* @deprecated 使用 --xxx 替代 */
   → 保留 Token 定义，不删除

2. 迁移引用
   → 全局搜索引用处，替换为新 Token
   → 提交迁移 PR

3. 确认迁移完成
   → 确认无代码引用废弃 Token
   → 下个大版本删除废弃 Token 定义
```

### 15.4 版本管理

| 版本类型 | 变更内容 | 示例 |
|----------|----------|------|
| Major (v2.0) | Token 架构变更、大量 Token 废弃/重命名 | CSS Variable 前缀变更 |
| Minor (v1.x) | 新增 Token、新增 Mixin、新增组件样式 | 新增 `--c-success` Token |
| Patch (v1.x.x) | Token 值微调、Mixin 修复、样式 bug 修复 | 调整 `--c-primary-hover` 色值 |

## 十八、Design Library 集成指南

### 18.1 Design Library 概述

项目已建立 TRAE Design Design Library（胶东收藏，ID: VC3AI487-C81IC），作为设计 token 和组件定义的权威来源。Design Library 与本文档的关系：

| 维度 | Design Library | 本文档（样式系统） |
|------|---------------|-------------------|
| Token 定义 | `colors_and_type.css` 中的 CSS Variables（权威来源） | `:root` 定义（工程实现参考） |
| Token 格式 | JSON (`css.json`) + CSS | CSS + SCSS |
| 组件定义 | `components/*.json` 契约文件 + `preview/component-*.html` 预览 | SCSS Mixin + BEM 样式模板 |
| 命名规范 | `jd-` BEM 前缀（jd-btn, jd-artifact-card） | 无前缀 BEM（collection-card, primary-btn） |
| 用途 | 设计工具集成、组件预览、Token 管理 | 前端工程实现、样式组织、构建流程 |

### 18.2 变量名对照

Design Library 使用 `--{scale}-{name}` 格式（如 `--navy-500`），前端工程使用语义化别名（如 `--c-primary`）。两套命名通过 semantic aliases 桥接：

```css
/* Design Library 语义别名（已在 colors_and_type.css 中定义） */
--color-primary: var(--navy-500);           /* 对应工程层 --c-primary */
--color-primary-hover: #163066;            /* 对应工程层 --c-primary-hover */
--color-on-surface: var(--text-primary-700); /* 对应工程层 --c-text */
--color-muted-foreground: var(--text-primary-400); /* 对应工程层 --c-text-secondary */
--color-background: var(--surface-50);      /* 对应工程层 --c-white */
--color-surface-container-low: var(--surface-200); /* 对应工程层 --bg-band */
```

### 18.3 组件集成方式

Design Library 中的组件预览文件（`preview/component-*.html`）是可直接复制的 HTML 代码模板：

1. 复制预览文件中的 HTML 结构到 Vue 组件模板
2. 将 `jd-` 前缀类名替换为项目 BEM 类名（或保留 `jd-` 前缀作为统一组件类名）
3. 引入 `colors_and_type.css` 和 `components.css` 获取样式
4. 参考 `components/*.json` 契约文件了解支持的状态和变体

### 18.4 Token 更新流程

```
Token 更新标准流程（含 Design Library 同步）：

1. 提出变更 → Issue 中描述，标注 label: design-token-change

2. 评估影响
   → 全局搜索 var(--xxx) 确认引用处数量
   → 评估 Design Library colors_and_type.css 同步需求

3. 修改 Token（三处同步）
   → Design Library: colors_and_type.css + css.json
   → 工程层: shared/tokens/variables.css + variables.scss
   → 文档: 视觉风格文档 + 本文档

4. 更新组件契约
   → 如 Token 变更影响组件，同步更新 components/*.json
   → 更新组件预览 preview/component-*.html

5. 验证
   → Design Library 组件预览视觉走查
   → 前端项目视觉走查
   → 可访问性检测

6. 发布
   → Design Library metadata.json 版本号 +1
   → 提交 PR，关联 Issue
```
